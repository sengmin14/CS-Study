# 🥕 Singleton Pattern

- 메모리 낭비할 수 있는 이점이 있다.
```
사용자가 1초에 10번 똑같은 요청을 보내면 처리하기 위한 똑같은 객체를 1초에 10번 생성하고 소멿되는
메모리 낭비 문제가 발생하게 된다.
하지만 싱글톤 패턴을 사용하면 최초 한번 new로 객체를 생성하고 해당 객체를 이후에도 사용하도록
공유(static)하면 메모리 낭비 문제를 방지할 수 있다.
```
- 대표적으로 데이터베이스 연결 모듈을 예로 들 수 있다.
  - 데이터베이스에 접속하는 작업(I/O바운드)은 그 자체로 무거운 작업에 속하여 또한 한번만 객체를 생성하고
    돌려쓰면 되기에 굳이 여러번 생성할 필요가 없다.
  - 디스크 연결, 네트워크 통신, DBCP 컨넥션풀, 스레드풀, 캐시, 로그 기록 객체 등에 이용된다.

## Eager Initialization
- 한번만 미리 만들어두는, 가장 직관적이면서도 심플한 기법
- static fianl 이라 멀티 쓰레드 환경에서도 안전함
- 그러나 static 멤버는 당장 객체를 사용하지 않더라도 메모리에 적재하기 때문에 만일 리소스가 큰 객체일 경우,
  공간 자원 낭비가 발생함
- 예외 처리를 할 수 없음
